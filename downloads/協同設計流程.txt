`develop` 分支**需要事先建立**，否則你在執行 `git checkout develop` 時會收到錯誤訊息，表示該分支不存在。

### 🧱 建立 `develop` 分支的方式

如果你是專案的初始化者或管理者，可以從 `main` 分支建立 `develop` 分支：

```bash
git checkout -b develop main
git push origin develop
```

這兩行指令會：
- 從 `main` 分支建立一個名為 `develop` 的新分支
- 並將它推送到遠端 GitHub 儲存庫

在團隊成員能夠使用 `develop` 分支之前，**必須先將遠端儲存庫 clone 到本地端**。這是 Git 協作的第一步，否則後續的 `checkout` 或 `pull` 都無法執行。

---

### 👥 團隊成員如何使用 `develop` 分支

在 `develop` 分支已由管理者建立並推送至遠端儲存庫後，其他組員可依以下步驟開始協作：

```bash
# 1. 將遠端儲存庫 clone 到本地端
git clone https://github.com/your-org/project.git
cd project

# 2. 取得遠端分支資訊
git fetch origin

🧠 git fetch origin 是什麼意思？
這個指令的作用是：
- 從遠端儲存庫（通常叫 origin）取得所有分支的最新狀態
- 更新本地的「遠端追蹤分支」（例如 origin/develop, origin/main）
- 不會改變你目前工作的分支內容
它是「同步遠端狀態到本地」，但不會合併或覆蓋你的工作。

# 3. 切換到 develop 分支
git checkout develop

# 4. 同步最新內容
git pull origin develop
```
---

這樣的流程可以確保每位組員都在最新的 `develop` 分支上進行開發，避免版本落差與合併衝突。

合併後刪除遠端分支

git push origin --delete feature/component-b

清理掉本地儲存庫中已不存在的遠端分支資訊。

## 🧩 組員 A 的整合流程（合併 B、C 的 PR 並整合至 main）

### 🔹 1. 合併 B、C 的 PR 至 develop 分支

- 在 GitHub 上審查並合併 PR
- 或使用指令：

```bash
git checkout develop
git pull origin develop
git merge feature/component-b
git merge feature/component-c
```

### 🔹 2. 測試整合後的內容（Flask 編輯器）

```bash
flask --debug run
# 確認所有頁面與零件運作正常
```

### 🔹 3. 合併 develop 至 main 分支

```bash
git checkout main
git pull origin main
git merge develop
git push origin main
```

### 🔹 4. 清理分支（可選）

```bash
# 刪除本地分支
git branch -d feature/component-a
git branch -d feature/component-b
git branch -d feature/component-c

# 刪除遠端分支
git push origin --delete feature/component-a
git push origin --delete feature/component-b
git push origin --delete feature/component-c
```

## ✅ 額外建議：避免衝突與保持同步

- 每位組員在開始開發前，務必執行：
  ```bash
  git checkout develop
  git pull origin develop
  ```
- 若 develop 分支有更新，請在自己的功能分支中合併：
  ```bash
  git checkout feature/component-b
  git merge develop
  ```
